Cypher — Confidential Prediction Market Protocol
===================================================

A Solana program built with Anchor + Arcium for encrypted predictions.
Users place encrypted bets, Arcium circuits compute settlement without
revealing individual predictions, and winners claim payouts in USDC.

Architecture Overview
---------------------
Protocol Account Hierarchy:
  GlobalState (1 global PDA, seeds=["global_state"])
    └── Market (1 per question, seeds=["market", market_id])
         ├── LPPosition (creator's LP position, seeds=["lp-position", market, creator])
         ├── MarketVault (SPL token account, seeds=["market_vault", market])
         └── EncryptedPosition (per user per market, seeds=["position", market, user])

Plaintext Pools Model (post-refactor):
  Pools are stored as plaintext u64 in market.revealed_pool_0/1/2/3 and passed
  as plaintext arguments to Arcium circuits. The encrypted_pool_0/1/2/3 fields
  on Market are vestigial (zeroed at creation, never written by circuits).

  What stays encrypted (Enc<Shared, BetInput>):
    - User's bet amount (decrypted only inside Arcium)
    - User's side/outcome choice (decrypted only inside Arcium)

  What is plaintext:
    - Pool totals (revealed after each bet by circuit output)
    - Entry odds (computed in circuit, returned as plaintext)
    - Payout ratio (computed at resolution)
    - Outcome (confirmed by circuit, stored on Market)

Arcium-Specific Accounts:
  - MXEAccount: The Multi-Party Execution environment PDA for this program
  - ComputationDefinitionAccount: Pre-registered circuit bytecode (one per circuit type)
  - ArciumSignerAccount: PDA that signs for the program within Arcium (seeds=[SIGN_PDA_SEED])
  - MempoolAccount: Arcium's mempool where pending computations queue
  - ExecutingPool: Arcium's executing pool where active computations run
  - ComputationAccount: Represents a single computation job (derived from offset + MXE)
  - ClusterAccount: Arcium cluster state tracking nodes and rounds
  - FeePool: Arcium's fee pool for computation costs
  - ClockAccount: Arcium's clock for computation timing
  - AddressLookupTable: LUT for efficient address lookup in Arcium

Eight Arcium Circuits (4 YesNo + 4 MultiOutcome):
  YesNo circuits:
    place_private_bet_yesno:
      Input: (yes_pool: u64, no_pool: u64, net_amount_plaintext: u64,
              Enc<Shared, BetInput>)
      Output: (new_yes_pool: u64, new_no_pool: u64, entry_odds: u64)
      Notes: safe_amount = (b.amount == net_amount_plaintext) ? b.amount : 0
             Prevents inflated encrypted amounts from corrupting pools (H-1).

    reveal_market_outcome_yesno:
      Input: (yes_pool: u64, no_pool: u64, outcome_value: u8)
      Output: (yes_pool: u64, no_pool: u64, payout_ratio: u64, outcome: u8)
      Notes: outcome_value=0/1 (NO/YES). Output includes outcome_value (field_3)
             to prevent pending_outcome overwrite race (M-3).

    compute_yesno_payout:
      Input: (Enc<Shared, BetInput>, outcome: u8, payout_ratio: u64,
              net_amount_plaintext: u64)
      Output: (payout: u64, is_winner: bool)
      Notes: capped_amount = min(pos.amount, net_amount_plaintext)
             is_winner = (pos.side == outcome)

    compute_yesno_refund:
      Input: (Enc<Shared, RefundInput>, net_amount_plaintext: u64)
      Output: refund_amount: u64

  MultiOutcome circuits (same pattern, 4 pools):
    place_private_bet_multi:
      Input: (p0/p1/p2/p3: u64, net_amount_plaintext: u64,
              Enc<Shared, BetInput>)
      Output: (p0/p1/p2/p3: u64, entry_odds: u64)

    reveal_market_outcome_multi:
      Input: (p0/p1/p2/p3: u64, outcome_value: u8)
      Output: (p0/p1/p2/p3: u64, payout_ratio: u64, outcome: u8)

    compute_multi_payout:
      Input: (Enc<Shared, BetInput>, outcome: u8, payout_ratio: u64,
              net_amount_plaintext: u64)
      Output: (payout: u64, is_winner: bool)

    compute_multi_refund:
      Input: (Enc<Shared, RefundInput>, net_amount_plaintext: u64)
      Output: refund_amount: u64


INSTRUCTIONS — Complete Reference with Accounts
================================================

1. INITIALIZE (initialize)
   Signature: initialize(ctx: Context<Initialize>, protocol_fee_rate: u16, lp_fee_rate: u16)

   Purpose:
     One-time admin setup. Creates the GlobalState PDA that holds protocol-wide
     configuration: fee rates, USDC mint, treasury address, and a market counter
     used to assign unique market IDs. Must be called once before any markets.

   Accounts:
     admin (Signer, mut)
       - The transaction signer who pays for GlobalState initialization.
       - Stored as admin in GlobalState for future admin-only operations.
     global_state (Box<Account<GlobalState>>, init, payer=admin, seeds=["global_state"])
       - The protocol's single global config PDA being created here.
       - Stores fee rates, treasury, accepted mint, admin, and market counter.
     protocol_treasury (UncheckedAccount)
       - Wallet address that receives protocol fees from every bet.
       - Stored in GlobalState for all future fee-sending instructions.
     accepted_mint (InterfaceAccount<Mint>)
       - The SPL token mint that all markets use (e.g., USDC).
       - Stored in GlobalState to validate user deposits across instructions.
     system_program (Program<System>)
       - Required by Anchor for the PDA initialization (CPI to create account).

   Constraints:
     protocol_fee_rate <= 100 (1% max protocol fee)
     lp_fee_rate <= 500 (5% max LP fee)


2. REGISTER COMPUTATION DEFINITIONS (8 instructions, 4 YesNo + 4 Multi)
   init_place_bet_yesno_comp_def, init_reveal_yesno_comp_def,
   init_payout_yesno_comp_def, init_refund_yesno_comp_def,
   init_place_bet_multi_comp_def, init_reveal_multi_comp_def,
   init_payout_multi_comp_def, init_refund_multi_comp_def

   Signature: fn_name(ctx: Context<...>)

   Purpose:
     Each of the eight Arcium circuits must be registered on-chain before they
     can be used. Registers the .arcis circuit bytecode with the Arcium cluster,
     creating a ComputationDefinitionAccount PDA that the queue_computation
     macros reference via a fixed offset constant.

   Accounts (identical pattern for all eight, differing only in circuit name
            and comp_def_offset constant):
     payer (Signer, mut)
       - Pays for the computation definition account rent.
     mxe_account (Box<Account<MXEAccount>>, mut, address=derive_mxe_pda!())
       - The MXE execution environment PDA for this program.
     comp_def_account (UncheckedAccount, mut)
       - The ComputationDefinition PDA being created/initialized.
     address_lookup_table (UncheckedAccount, mut,
                           address=derive_mxe_lut_pda!(mxe_account.lut_offset_slot))
       - Arcium's Address Lookup Table for efficient account referencing.
     lut_program (UncheckedAccount, address=LUT_PROGRAM_ID)
       - The official Address Lookup Table program ID.
     arcium_program (Program<Arcium>)
       - The Arcium program that handles computation definition registration.
     system_program (Program<System>)
       - Required for any PDA creation CPIs.


3. CREATE MARKET (create_market)
   Signature: create_market(ctx: Context<CreateMarket>, question: String,
                            close_time: i64, category: u8)

   Purpose:
     Creator pays a $10 USDC bond (locked in MarketVault) and creates a Market
     PDA with the question text, close time, category, and default deadlines.
     The bond incentivizes honest behavior — if the creator fails to resolve
     in time, the bond can be forfeited. Also initializes an LPPosition to
     track the creator's LP fees earned from bets on this market.

   Accounts:
     creator (Signer, mut)
       - Market creator, pays the $10 bond from their token account.
       - Stored as both creator and resolver on the Market.
     global_state (Box<Account<GlobalState>>, mut, seeds=["global_state"])
       - Read to get the current market_counter for the new market's ID.
       - Mutable because market_counter is incremented.
     market (Box<Account<Market>>, init, payer=creator, space=MARKET_SPACE,
             seeds=["market", global_state.market_counter.to_le_bytes()])
       - The Market PDA being created. Seeds use the monotonically increasing
         counter so market IDs are sequential and deterministic.
       - market_type = 0 (YESNO), category set from parameter.
       - resolution_deadline = close_time + 7 days.
       - claim_deadline = 0, refund_deadline = 0 (set at resolution).
     lp_position (Box<Account<LPPosition>>, init, payer=creator,
                  seeds=["lp-position", market, creator])
       - Tracks the creator's LP position.
       - Note: fees_earned field is NOT incremented by place_bet instructions.
         LP fees accumulate on market.accumulated_lp_fees instead.
     market_vault (Box<Account<TokenAccount>>, init, payer=creator,
                   token::mint=accepted_mint, token::authority=market_vault,
                   seeds=["market_vault", market])
       - SPL token account that holds all USDC for this market.
       - Authority is set to itself (the vault PDA), so transfers out require
         PDA seeds signature.
     creator_token_account (Box<Account<TokenAccount>>, mut,
                            constraint: owner=creator, mint=accepted_mint)
       - Creator's personal USDC token account.
       - $10 bond is transferred FROM this account TO the market_vault.
     accepted_mint (InterfaceAccount<Mint>)
       - Validates that the vault uses the correct USDC mint.
     token_program (Program<Token>)
       - SPL Token program for the bond transfer and vault creation.
     system_program (Program<System>)
       - Required by Anchor for PDA and token account initialization.

   Parameters:
     question: String (max 200 bytes)
     close_time: i64 (Unix timestamp, must be in the future)
     category: u8 (0=Crypto, 1=Politics, 2=Sports, 3=Tech, 4=Economy,
                   5=Culture, 6=Beyond)

   Emits: MarketCreatedEvent


4. CANCEL MARKET (cancel_market)
   Signature: cancel_market(ctx: Context<CancelMarket>)

   Purpose:
     Creator can cancel a market before any bets are placed. The $10 bond is
     returned to the creator. Market must be Active and have 0 total bets.
     Market state is set to MARKET_STATE_CLOSED (1) to prevent further
     interaction.

   Accounts:
     creator (Signer, mut)
       - Must match market.creator. Receives the bond back.
     market (Box<Account<Market>>, mut, seeds=["market", market.market_id],
             constraint: creator==creator)
       - The market being cancelled. Must be ACTIVE with total_bets_count == 0.
       - state set to MARKET_STATE_CLOSED.
     market_vault (Box<Account<TokenAccount>>, mut, seeds=["market_vault", market])
       - Holds the creator's bond. PDA signs to transfer bond back.
     lp_position (Box<Account<LPPosition>>, mut,
                  seeds=["lp-position", market, creator])
       - LP position exists for PDA derivation validation.
     creator_token_account (Box<Account<TokenAccount>>, mut,
                            constraint: owner==creator)
       - Receives the $10 bond back from market_vault.
     token_program (Program<Token>)
       - SPL Token program for the transfer CPI.

   Emits: MarketCancelledEvent


5. WITHDRAW CREATOR FUNDS (withdraw_creator_funds)
   Signature: withdraw_creator_funds(ctx: Context<WithdrawCreatorFunds>)

   Purpose:
     After a market is resolved, the creator can withdraw their $10 bond plus
     any LP fees earned from user bets. LP fees are read from
     market.accumulated_lp_fees (NOT from lp_position.fees_earned). Market
     must be in RESOLVED state — bond is forfeited if market is unresolved.

   Accounts:
     creator (Signer, mut)
       - Must match market.creator. Receives bond + LP fees.
     market (Box<Account<Market>>, mut, seeds=["market", market.market_id],
             constraint: creator==creator)
       - Must be in RESOLVED state. bond_withdrawn set to true to prevent
         double-claim.
     lp_position (Box<Account<LPPosition>>, mut,
                  seeds=["lp-position", market, creator])
       - Marked fees_claimed = true, records fees_claimed_amount.
     market_vault (Box<Account<TokenAccount>>, mut, seeds=["market_vault", market])
       - PDA signs to transfer bond + LP fees to creator.
     creator_token_account (Box<Account<TokenAccount>>, mut,
                            constraint: owner==creator)
       - Receives the total (bond + LP fees).
     token_program (Program<Token>)

   Emits: CreatorWithdrawnEvent


6. ADMIN CLAIM REMAINING (admin_claim_remaining)
   Signature: admin_claim_remaining(ctx: Context<AdminClaimRemaining>)

   Purpose:
     Sweeps unclaimed USDC from a market vault to the protocol treasury after
     all claim/refund periods have expired. Uses state-conditional time guards
     (C-2 fix):
       - For resolved markets: must be past refund_deadline.
       - For unresolved/active markets: must be past
         resolution_deadline + DEFAULT_REFUND_PERIOD (14 days).

   Accounts:
     admin (Signer, mut)
       - Must match global_state.admin. Authenticates as protocol admin.
     global_state (Box<Account<GlobalState>>, seeds=["global_state"],
                   constraint: admin==admin)
       - Read to get protocol_treasury address for transfer destination.
     market (Box<Account<Market>>, mut, seeds=["market", market.market_id])
       - Checked to ensure admin_claimed_remaining is not already true.
       - State used to determine deadline check (resolved vs. unresolved).
     market_vault (Box<Account<TokenAccount>>, mut,
                   seeds=["market_vault", market])
       - Source of remaining funds. PDA signs to release funds.
     protocol_treasury (Box<Account<TokenAccount>>, mut,
                        constraint: key==global_state.protocol_treasury)
       - Destination for the sweep.
     token_program (Program<Token>)


7. PLACE PRIVATE BET YESNO (place_private_bet_yesno)
   Signature: place_private_bet_yesno(ctx, computation_offset: u64,
               bet_amount_usdc: u64, encrypted_amount: [u8;32],
               encrypted_side: [u8;32], pub_key: [u8;32], nonce: u128)

   Purpose:
     User places an encrypted bet on a Yes/No market. The bet amount and side
     are client-side encrypted before submission. The instruction:
       1. Validates market is YESNO type, ACTIVE state, before close_time.
       2. Computes protocol fee (sent to treasury) and LP fee (accrued to
          market.accumulated_lp_fees).
       3. Transfers bet amount from user -> market_vault.
       4. Sends protocol fee from vault -> treasury immediately.
       5. Creates EncryptedPosition PDA storing encrypted payload + net_amount.
       6. Queues Arcium computation (place_private_bet_yesno) with:
          - Plaintext revealed_pool_0/1 (pool totals)
          - Plaintext net_amount (H-1: prevents encrypted amount mismatch)
          - Encrypted amount + side under Enc<Shared, BetInput>
       7. Callback updates revealed_pool_0/1, total_bets_count, entry_odds.

   Fee model:
     protocol_fee = bet_amount_usdc * protocol_fee_rate / 10_000
     lp_fee = bet_amount_usdc * lp_fee_rate / 10_000
     net_amount = bet_amount_usdc - protocol_fee - lp_fee

   The circuit asserts safe_amount = (b.amount == net_amount) ? b.amount : 0,
   preventing an attacker from encrypting a different amount than what was
   actually deposited on-chain (H-1 fix).

   Accounts (Cypher custom accounts):
     user (Signer)
       - The bettor. Signed to authorize USDC transfer.
     global_state (Box<Account<GlobalState>>, seeds=["global_state"])
       - Read-only: provides accepted_mint, fee rates, protocol_treasury.
     market (Box<Account<Market>>, mut, seeds=["market", market.market_id],
             constraint: market_type==YESNO)
       - Must be ACTIVE before close_time.
       - revealed_pool_0/1 updated in callback.
     market_vault (Box<Account<TokenAccount>>, mut,
                   seeds=["market_vault", market])
       - Receives the user's bet. Intermediate for protocol fee transfer.
     user_token_account (Box<Account<TokenAccount>>, mut,
                         constraint: owner==user, mint==accepted_mint)
       - User's USDC token account. Source of the bet amount.
     protocol_treasury (Box<Account<TokenAccount>>, mut,
                        constraint: key==global_state.protocol_treasury)
       - Receives the protocol fee portion.
     position (Box<Account<EncryptedPosition>>, init, payer=payer,
               seeds=["position", market, user])
       - User's position PDA. Stores encrypted payload, user_pubkey, nonce,
         net_amount. entry_odds written in callback.
     token_program (Program<Token>)
       - SPL Token program for USDC transfers.

   Arcium accounts (standard for all queue_computation instructions):
     payer (Signer, mut)
       - Pays for Arcium computation and position PDA rent.
     sign_pda_account (Box<Account<ArciumSignerAccount>>, init_if_needed,
                       seeds=[SIGN_PDA_SEED])
       - Program's signer PDA within Arcium.
     mxe_account (Box<Account<MXEAccount>>, address=derive_mxe_pda!())
       - The MXE execution environment.
     mempool_account (UncheckedAccount, mut, derive_mempool_pda)
       - Computation mempool.
     executing_pool (UncheckedAccount, mut, derive_execpool_pda)
       - Executing pool.
     computation_account (UncheckedAccount, mut,
                          derive_comp_pda(computation_offset, mxe))
       - Specific computation slot for this bet.
     comp_def_account (derive_comp_def_pda(PLACE_BET_YESNO))
       - Pre-registered circuit definition.
     cluster_account (mut, derive_cluster_pda(mxe))
       - Arcium cluster state.
     pool_account (mut, ARCIUM_FEE_POOL)
       - Fee pool for computation costs.
     clock_account (mut, ARCIUM_CLOCK)
       - Arcium timing clock.
     system_program (Program<System>)
       - Required for PDA initialization.
     arcium_program (Program<Arcium>)
       - Receives the queue_computation CPI.

   ArgBuilder order:
     .plaintext_u64(revealed_pool_0)
     .plaintext_u64(revealed_pool_1)
     .plaintext_u64(net_amount)           ← H-1: verified on-chain amount
     .x25519_pubkey(pub_key)
     .plaintext_u128(nonce)
     .encrypted_u64(encrypted_amount)
     .encrypted_u8(encrypted_side)
     .build()


8. PLACE BET YESNO CALLBACK (place_private_bet_yesno_callback)
   #[arcium_callback(encrypted_ix = "place_private_bet_yesno")]

   Purpose:
     Called by Arcium after place_private_bet_yesno circuit completes.
     Verifies ZK proof, then:
       - Updates market.revealed_pool_0/1 with new pool totals.
       - Increments market.total_bets_count.
       - Writes entry_odds to user's EncryptedPosition.

   Accounts:
     arcium_program, comp_def_account, mxe_account, computation_account,
     cluster_account, instructions_sysvar
       - Standard Arcium callback verification (seed-constrained).
     market (Box<Account<Market>>, mut) — Custom
       - revealed_pool_0/1 updated. total_bets_count incremented.
       - Not seed-constrained (passed by Arcium from queue time).
     position (Box<Account<EncryptedPosition>>, mut) — Custom
       - entry_odds written.

   Emits: BetPlacedEvent


9. RESOLVE MARKET YESNO (resolve_market_yesno)
   Signature: resolve_market_yesno(ctx, computation_offset: u64,
                                   outcome_value: u8)

   Purpose:
     The resolver submits the outcome. outcome_value is 0 (NO) or 1 (YES),
     matching the BetInput.side encoding (C-1 fix). Outcome is stored as
     pending_outcome, then Arcium computation is queued that reveals the
     winning pool total and computes payout_ratio. The callback uses the
     circuit's output outcome (field_3) instead of pending_outcome to
     prevent the pending_outcome overwrite race (M-3 fix).

   Accounts (13 total):
     resolver (Signer)
       - Must match market.resolver.
     market (Box<Account<Market>>, mut, seeds=["market", market.market_id],
             constraint: market_type==YESNO, resolver==resolver)
       - Must be ACTIVE and past close_time.
       - market.constraint includes market_type check (M-2).
     (Standard Arcium queue accounts — see #7 for details)
     comp_def_account uses COMP_DEF_OFFSET_REVEAL_YESNO.

   ArgBuilder order:
     .plaintext_u64(revealed_pool_0)
     .plaintext_u64(revealed_pool_1)
     .plaintext_u8(outcome_value)  ← 0=NO, 1=YES
     .build()

   Constraints:
     require!(outcome_value == 0 || outcome_value == 1)  ← C-1: was 1/2, now 0/1


10. REVEAL OUTCOME YESNO CALLBACK (reveal_market_outcome_yesno_callback)
    #[arcium_callback(encrypted_ix = "reveal_market_outcome_yesno")]

    Purpose:
      Called by Arcium after reveal circuit completes. Verifies ZK proof, then:
        - Writes revealed_pool_0 (YES total) and revealed_pool_1 (NO total).
        - Writes payout_ratio (scaled by 1e9).
        - Sets market.state = RESOLVED (2).
        - Sets market.outcome = o.field_3 (circuit output, NOT pending_outcome).
        - Sets resolution_time, claim_deadline, refund_deadline timestamps.
        - Guard: require!(state != RESOLVED) prevents double-resolution (M-3).

    Accounts:
      arcium_program, comp_def_account, mxe_account, computation_account,
      cluster_account, instructions_sysvar
        - Standard Arcium callback verification.
      market (Box<Account<Market>>, mut) — Custom
        - Updated with all resolved state.

    Emits: MarketResolvedEvent


11. CLAIM PAYOUT YESNO (claim_payout_yesno)
    Signature: claim_payout_yesno(ctx, computation_offset: u64)

    Purpose:
      A winning user claims their payout after the market is resolved. Queues
      an Arcium computation (compute_yesno_payout circuit) that decrypts the
      user's encrypted position, checks if side matches outcome, and computes
      payout = capped_amount * payout_ratio / 1e9. The callback:
        - Guards against double-claim with !claimed check (H-2).
        - Transfers payout from market_vault -> user's token account.
        - Caps payout using net_amount_plaintext position.net_amount (H-1).

    Queue accounts (15 total):
      user (Signer)
        - Must match position.user.
      market (Box<Account<Market>>, seeds=["market", market.market_id])
        - Must be RESOLVED. Read for payout_ratio, outcome, claim_deadline.
      position (Box<Account<EncryptedPosition>>, mut,
                seeds=["position", market, user], constraint: user==user)
        - Must be unclaimed. encrypted_amount, encrypted_side, user_pubkey,
          nonce, net_amount read and passed to circuit.
      market_vault (Box<Account<TokenAccount>>, mut,
                    seeds=["market_vault", market])
        - PDA signs to release payout funds in callback.
      user_token_account (Box<Account<TokenAccount>>, mut,
                          constraint: owner==user)
        - Destination for payout in callback.
      token_program (Program<Token>)
        - SPL Token program for the transfer CPI.
      (Standard Arcium queue accounts: payer, sign_pda, mxe, mempool, exec,
       computation, comp_def (PAYOUT_YESNO), cluster, fee pool, clock,
       system, arcium)

    ArgBuilder order:
      .x25519_pubkey(position.user_pubkey)
      .plaintext_u128(position.nonce)
      .encrypted_u64(position.encrypted_amount)
      .encrypted_u8(position.encrypted_side)
      .plaintext_u8(market.outcome)
      .plaintext_u64(market.payout_ratio)
      .plaintext_u64(position.net_amount)   ← H-1: caps the payout
      .build()


12. PAYOUT YESNO CALLBACK (compute_yesno_payout_callback)
    #[arcium_callback(encrypted_ix = "compute_yesno_payout")]

    Purpose:
      Called by Arcium after payout circuit completes. Verifies ZK proof, then:
        - Guards: require!(!position.claimed) prevents double-pay (H-2).
        - Reads payout_amount and is_winner from circuit output.
        - If is_winner and payout > 0: transfers market_vault -> user.
        - Increments market.total_payouts_claimed.
        - Marks position.claimed = true.

    Accounts:
      arcium_program, comp_def_account, mxe_account, computation_account,
      cluster_account, instructions_sysvar
        - Standard Arcium callback verification.
      position (Box<Account<EncryptedPosition>>, mut) — Custom
        - Marked claimed = true.
      user (UncheckedAccount, mut) — Custom
        - User wallet for event emission.
      market (Box<Account<Market>>, mut) — Custom
        - total_payouts_claimed incremented. vault_bump read for PDA signing.
      market_vault (Box<Account<TokenAccount>>, mut) — Custom
        - Source of payout funds. PDA signs the transfer.
      user_token_account (Box<Account<TokenAccount>>, mut) — Custom
        - Destination for the payout.
      token_program (Program<Token>) — Custom
      system_program (Program<System>) — Custom

    Emits: PayoutClaimedEvent


13. CLAIM REFUND YESNO (claim_refund_yesno)
    Signature: claim_refund_yesno(ctx, computation_offset: u64)

    Purpose:
      If the resolution deadline passes without the market being resolved,
      any bettor can claim a refund of their stake. Queues compute_yesno_refund
      circuit which decrypts the position, caps to net_amount_plaintext (C-3).
      Callback transfers refund from market_vault -> user.

    Conditions:
      - market state must NOT be RESOLVED.
      - Clock must be past resolution_deadline.
      - position must be unclaimed.

    Accounts (15 total — same structure as claim_payout_yesno):
      user, market, position — same as claim_payout.
      market_vault, user_token_account, token_program
        - Included in queue context for callback transfer preparation.
      (Standard Arcium queue accounts with REFUND_YESNO comp def offset)

    ArgBuilder order:
      .x25519_pubkey(pubkey)
      .plaintext_u128(nonce)
      .encrypted_u64(encrypted_amount)
      .plaintext_u64(net_amount)   ← C-3: caps refund to verified on-chain amount
      .build()


14. REFUND YESNO CALLBACK (compute_yesno_refund_callback)
    #[arcium_callback(encrypted_ix = "compute_yesno_refund")]

    Purpose:
      Called by Arcium after refund circuit completes. Verifies ZK proof,
      reads refund_amount from circuit output (capped to net_amount by
      circuit), transfers from vault -> user. Marks position.claimed = true.
      Guards against double-refund (H-2).

    Accounts (same as payout callback #12):
      arcium_program, comp_def_account, mxe_account, computation_account,
      cluster_account, instructions_sysvar

Account Usage Patterns — Why So Many Accounts?
===============================================

The Arcium integration is the primary reason for the high account counts.
There are three distinct account categories:

A) Cypher Protocol Accounts (market, vault, positions, fees)
   - These carry the core business logic: market state, user positions, fees.
   - Seed-constrained to prevent account substitution attacks.
   - Market accounts use seeds=["market", market_id] so anyone can derive them.
   - Positions use seeds=["position", market, user] for per-user uniqueness.
   - Market constraints include market_type checks (M-2 fix).

B) Arcium Computation Accounts (MXE, mempool, exec pool, comp, cluster, etc.)
   - Required whenever queue_computation() is called.
   - comp_def_account links to a specific pre-registered circuit.
   - computation_account is derived from (offset, MXE) to prevent ID collisions.
   - sign_pda_account gives the program authority within Arcium.

C) Token Handling Accounts (vaults, user token accounts, treasury)
   - Required for USDC transfers via SPL Token CPI.
   - market_vault PDA is both the escrow and the signing authority.
   - Protocol treasury is validated against GlobalState.

Why PDA seeds matter:
  - Prevent "rogue account" attacks.
  - market_vault seeds=["market_vault", market] binds vault to its market.

Why callbacks use fewer constraints:
  - Callback accounts are not seed-constrained because they are passed by the
    Arcium cluster from the original queue_computation invocation. Seed
    verification happens at queue time, not callback time.


Lifecycle Summary
=================

Market:     Created -> Active -> Resolved -> (payouts/refunds claimed)
            Active -> Closed (cancelled, if 0 bets)
            Active -> Unresolved (deadline passed) -> refunds

Position:   Created (at bet) -> Claimed (at payout/refund)

LPPosition: Created (at market) -> fees accrue on Market.accumulated_lp_fees
                                          -> Withdrawn (after resolution)

Bond:       Paid ($10 USDC at market creation) -> Returned (after resolution)
                                                     -> Forfeited (if unresolved)

Deadlines:
  close_time              — when betting stops (set at creation)
  resolution_deadline     — close_time + 7 days (creator must resolve by)
  claim_deadline          — resolution_time + 14 days (set at resolution)
  refund_deadline         — resolution_time + 28 days (set at resolution)
  (For unresolved markets: refund available after resolution_deadline)


State Machine
=============

  MARKET_STATE_ACTIVE = 0      -> MARKET_STATE_RESOLVED = 2 (via resolve callback)
                                -> MARKET_STATE_CLOSED = 1 (via cancel, if 0 bets)
                                -> MARKET_STATE_UNRESOLVED = 3 (if resolution_deadline passes without resolve)
                                   (Note: MARKET_STATE_UNRESOLVED is defined but not explicitly set — state remains ACTIVE;
                                    refund instructions check !RESOLVED + past resolution_deadline)

  State transitions enforced by:
    create_market:       sets MARKET_STATE_ACTIVE
    cancel_market:       sets MARKET_STATE_CLOSED (requires ACTIVE + 0 bets)
    resolve callback:    sets MARKET_STATE_RESOLVED (guards against double-resolve M-3)
    withdraw_creator:    requires MARKET_STATE_RESOLVED
    claim_payout:        requires MARKET_STATE_RESOLVED
    claim_refund:        requires !MARKET_STATE_RESOLVED + past resolution_deadline
    admin_claim_remaining: state-conditional deadline check (C-2 fix)


Security Fixes Applied
======================

C-1: YesNo outcome encoding normalized (0=NO, 1=YES) to match BetInput.side.
     resolve_market_yesno: require!(outcome_value == 0 || outcome_value == 1).

C-2: admin_claim_remaining: state-conditional time guards.
     Resolved markets: must be past refund_deadline.
     Unresolved markets: must be past resolution_deadline + 14 days.

H-1: net_amount_plaintext passed to place_bet and payout circuits.
     Circuit enforces: safe_amount / capped_amount <= net_amount_plaintext.
     Prevents inflated encrypted amounts from corrupting pools or payouts.

H-2: !position.claimed guard at the start of all payout/refund callbacks.
     Prevents double-claim from concurrent queue_computation calls.

M-1: cancel_market now sets market.state = MARKET_STATE_CLOSED.

M-2: All queue instructions now include market_type constraints
     (require!(market_type == YESNO) or require!(market_type == MULTIOUTCOME)).

M-3: Reveal callbacks use circuit output (field_3/field_5) for outcome
     instead of pending_outcome. Double-resolution guard prevents overwrite.
     Outcome encoded in circuit output: (..., outcome_value.reveal()).

L-1: user_token_account owner constraint in payout callback.

C-3: Refund circuits cap with net_amount_plaintext (same H-1 pattern).


Known Issues / Audit Findings
=============================

1. Encrypted BetInput round-trip (Finding 1 in yes_no_total.md):
   The client encrypts [amount, side] as a tuple with cipher.encrypt(), but
   the circuit receives them via separate .encrypted_u64/.encrypted_u8
   ArgBuilder calls. The ciphertext layout may not match Enc<Shared, BetInput>
   expectations, causing garbage decryption. Symptom: all bets land in pool_3,
   entry_odds = 1.0, winners get 0 payout. Fix: split into two separate
   Enc<Shared, u64> + Enc<Shared, u8> parameters, or match the exact byte
   layout.

2. b.side wildcard arm (Finding 3):
   Multi circuits accept any b.side via wildcard `_` arm -> pool_3.
   Markets with num_outcomes < 4 silently accept and orphan bets with
   side >= num_outcomes. Mitigation: document as caller responsibility;
   or enforce side bounds inside circuit.

3. LPPosition.fees_earned never written:
   LP fees accumulate on market.accumulated_lp_fees in both YesNo and
   Multi place_bet instructions. The lp_position.fees_earned field is
   initialized to 0 and never incremented. LPPosition is only used for
   PDA derivation and fees_claimed tracking.


MULTI-OUTCOME INSTRUCTIONS SUMMARY
==================================

The Multi-Outcome protocol supports markets with 2-4 discrete outcomes.
Follows the same architecture as Yes/No but with 4 encrypted pools and
side/outcome = 0-3 instead of 0/1.

Registered as 8 comp defs (4 per market type).

Aspect                        | Yes/No                          | Multi-Outcome
───────────────────────────── | ─────────────────────────────── | ─────────────────────────
market_type                   | 0 (YESNO)                       | 1 (MULTIOUTCOME)
num_outcomes                  | 0 (unused)                      | 2, 3, or 4
side encoding                 | 1=YES, 0=NO                     | 0, 1, 2, or 3
Revealed pools                | revealed_pool_0/1               | revealed_pool_0/1/2/3
lp_position in place_bet      | Not in context (LP fees on      | Not in context (LP fees on
                              | market.accumulated_lp_fees)     | market.accumulated_lp_fees)
claim_payout queue accounts   | Includes token accounts         | Includes token accounts
Payout circuit input enc      | Enc<Shared, BetInput>           | Enc<Shared, BetInput>
Comp def offsets              | PLACE_BET_YESNO, etc.           | PLACE_BET_MULTI, etc.
