CREATOR

The four circuit that actually runs ?

Circuit 1: place_private_bet_yesno
  ← runs at bet time (once per user)
  ← encrypted_pool_0 (YES) + encrypted_pool_1 (NO) updated inside MXE
  ← entry_odds locked for this user, stored PUBLIC on position

Circuit 2: reveal_market_outcome_yesno
  ← runs once at resolution
  ← decrypts both pools, returns PLAINTEXT yes_pool, no_pool, payout_ratio
  ← after this: everyone can see YES=$300, NO=$50

Circuit 3: compute_yesno_payout
  ← runs once per user who claims
  ← decrypts their side + amount, checks against outcome
  ← returns PLAINTEXT payout + is_winner
  ← callback directly transfers USDC

Circuit 4: compute_yesno_refund
  ← only if creator never resolves within 7 days
  ← decrypts amount, returns it as plaintext refund
  ← callback directly transfers USDC back
